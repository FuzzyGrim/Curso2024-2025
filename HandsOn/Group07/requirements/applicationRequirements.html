Application Requirements for Smart City Lighting System

Objective

The goal of this application is to create a monitoring system for public lighting in Smart Cities, using the Alumbrado PÃºblico dataset. The system will allow users to view and analyze the status of streetlights and optimize energy consumption.

Features

  • Real-time Monitoring: The system will provide real-time updates on the status of each streetlight in the city.
  • Energy Consumption Analysis: Users will be able to track energy usage by each streetlight and optimize consumption based on location and time of day.
  • Geographical Mapping: The application will include a map interface that shows the location of each streetlight with color-coded indicators for their status.

User Interface Mock-ups

The application will include the following user interfaces:

  • Dashboard: Overview of energy consumption and lighting statuses.
  • Map View: Interactive map showing the location of streetlights with detailed information when clicked.
  • Reports: Exportable reports on energy consumption, streetlight status, and maintenance needs.

Technical Requirements

  • Backend: The backend will be built using Node.js and will consume the CSV dataset, converting it into RDF for semantic queries.
  • Frontend: The frontend will be built using React and integrated with map services like Google Maps for displaying geospatial data.
  • Database: The application will use a triple store (e.g., Apache Jena) for storing and querying RDF data.
